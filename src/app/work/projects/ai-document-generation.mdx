## Overview

I built an AI system that generates personalized insurance documents (policies,
endorsements, claims reports) from structured client data and natural language
inputs. This replaced a slow manual process that relied on copy-pasting and Word
templates.

## Key Features

- **Natural Language Input**: Users describe needs in plain English; AI
  generates full documents.
- **Template Engine with AI**: Combines predefined legal templates with dynamic
  AI content generation.
- **Compliance & Review**: Automatic flagging of sensitive clauses and version
  control.
- **PDF Export + e-Signature Integration**: Ready-to-send documents.

## Technologies Used

- **AI**: OpenAI GPT-4o + prompt engineering
- **Backend**: Node.js, TypeScript, Supabase
- **Frontend**: React + Next.js
- **Document Generation**: PDF-lib, React-PDF
- **Automation**: n8n for data ingestion

## Challenges and Learnings

Legal accuracy was critical. I implemented a two-stage generation process
(draft + validation) and created a review interface for quick human approval.
Learned how to structure prompts for consistent, professional tone in regulated
industries.

## Outcome

- **80% faster** document creation (hours → minutes)
- Significant reduction in human error and compliance risk
- Improved client experience with faster turnaround
- Enabled the company to scale operations without hiring more underwriters
