## Overview

I designed and implemented a complete AI automation system for a shared Azure
inbox that receives high volumes of business proposals and miscellaneous emails.
The system automatically classifies emails, extracts structured project data,
calculates taxes based on location, stores everything in a database, and
organizes attachments in SharePoint, eliminating the need for manual review in
most cases.

## Key Features

- **Smart Email Classification**: Determines whether an email is a business
  proposal using optimized rules (regex + whitelists/blacklists) before sending
  to OpenAI.
- **Structured Data Extraction**: Uses OpenAI to extract project details, full
  address, and relevant information from emails and attachments.
- **Automated Tax Calculation**: Determines applicable taxes based on address
  and ZIP+4.
- **SharePoint Organization**: Automatically creates project folders and uploads
  all attachments.
- **Database Integration**: Saves all structured data into PostgreSQL (Supabase)
  for further processing.
- **Real-time Dashboard**: Built with Next.js to monitor processed emails, view
  results, handle exceptions, and search historical data.
- **Smart Tagging & Auditing**: All emails are tagged (Processed, Error,
  Non-Proposal) with full audit trail.

## Technologies Used

- **Automation Engine**: n8n (multiple workflows with loops and error handling)
- **AI**: OpenAI (GPT-4o) for data extraction and complex reasoning
- **Frontend**: Next.js, React, TypeScript, Tailwind
- **Database**: Supabase (PostgreSQL)
- **Integrations**: Microsoft Graph API, SharePoint, Azure App Permissions
- **Infrastructure**: Self-hosted n8n, cron jobs, flag system with TTL

## Challenges and Learnings

- The inbox was shared, so I couldn't use standard Outlook nodes. I had to build
  everything using Microsoft Graph API with HTTP requests and application-level
  permissions in Azure.
- To keep AI costs under control, I implemented an initial fast classification
  layer in JavaScript (regex + lists) instead of using OpenAI for every email.
- Processing large emails + attachments required careful workflow design with
  multiple stages and loops to avoid timeouts and rate limits.
- Created a custom flag system with TTL (time-to-live) to prevent duplicate
  SharePoint folders when workflows overlap or retry.

These technical constraints taught me how to build robust, cost-efficient, and
production-grade automation systems that can handle real enterprise
environments.

## Outcome

- **Processing capacity increased dramatically**: From dozens of emails per day
  manually to **200 emails in approximately 20 minutes**.
- **Massive time savings**: Reduced manual inbox processing by **~85-90%**.
- **Improved accuracy** and consistency in data extraction.
- Full visibility and control through the custom dashboard.
- Emails are automatically categorized and tagged for easy follow-up on
  exceptions.

This project is a clear example of how I combine n8n, OpenAI, and modern
full-stack tools to deliver high-impact automations that generate real ROI for
clients by replacing expensive manual labor with reliable intelligent systems.
